# BK-UI 작업 지침

이 문서는 저장소 전체에 적용한다. 하위 디렉터리에 더 구체적인 `AGENTS.md`가 있으면 해당 지침을 함께 따르며, 충돌하는 경우 하위 지침을 우선한다.

## React 기준

- React 관련 구현을 시작하기 전에 [React 공식 버전 페이지](https://react.dev/versions)와 공식 문서를 확인하고, npm의 `latest` 채널에 해당하는 최신 안정 버전을 기준으로 판단한다.
- 2026-07-13 기준 공식 문서의 최신 버전은 React 19.2이며, 최신 패치 릴리스는 19.2.7이다. 이 숫자를 영구적인 전제로 삼지 말고 작업 시점에 다시 확인한다.
- Canary와 Experimental API는 사용자가 명시적으로 요청하지 않는 한 사용하지 않는다.
- 저장소에 설치된 React 버전보다 새로운 API가 필요하면 먼저 호환성을 확인한다. 의존성 변경이 필요할 때는 `react`, `react-dom`, 관련 타입, 테스트 도구와 잠금 파일을 함께 정합성 있게 갱신하고 별도의 작업 및 커밋으로 분리한다.
- React의 최신 권장 패턴과 공식 API를 우선하고, 폐기 예정 API나 불필요한 레거시 패턴을 새 코드에 도입하지 않는다.

## 컴포넌트 설계

- shadcn/ui처럼 복사, 조합, 확장이 쉬운 컴포넌트를 지향한다. 특정 페이지나 데모의 데이터와 비즈니스 로직을 재사용 가능한 UI 컴포넌트 안에 고정하지 않는다.
- 큰 만능 컴포넌트보다 역할이 분명한 작은 primitive와 compound component를 조합한다. 상태, 표현, 도메인 데이터를 가능한 한 분리한다.
- 공개 API는 TypeScript 타입으로 명확하게 정의하고, 합리적인 기본값을 제공한다. 소비자가 `className`, 표준 HTML 속성, `aria-*`, `data-*`, 이벤트 핸들러를 전달하고 확장할 수 있게 한다.
- 스타일 변형과 상태는 예측 가능한 props 또는 `data-*` 속성으로 노출한다. 클래스 병합에는 저장소의 `clsx`와 `tailwind-merge` 조합을 사용해 소비자 스타일이 안전하게 확장되도록 한다.
- 접근성을 기본 동작으로 포함한다. 시맨틱 요소, 키보드 조작, 포커스 표시, 적절한 ARIA 이름을 구현하고 마우스 전용 인터랙션을 만들지 않는다.
- 재사용 가능한 로직은 hook이나 순수 함수로 분리하고, 데모·preview·mock 데이터는 컴포넌트 구현과 분리한다. 새 공개 컴포넌트에는 명확한 export 경로와 사용 예시를 제공한다.
- 동작 변경에는 해당 수준의 테스트를 추가하거나 갱신하고 `pnpm lint`, 관련 `pnpm test`, `pnpm build` 중 영향 범위에 필요한 검증을 실행한다.

## 커밋 규칙

- 작업을 완료하고 검증한 뒤 반드시 커밋한다. 사용자가 명시적으로 커밋하지 말라고 요청한 경우만 예외로 한다.
- 모든 커밋 제목은 `<prefix>: <summary>` 형식을 사용한다. 대표 prefix는 `feat`, `fix`, `refactor`, `test`, `docs`, `style`, `perf`, `chore`, `build`, `ci`다.
- 한 커밋에는 하나의 논리적 작업만 담는다. 서로 독립적으로 설명하거나 되돌릴 수 있는 두 가지 이상의 작업은 각각 구현, 검증, 커밋한다.
- 커밋 전 `git diff --staged`로 범위를 확인하고 현재 작업에 필요한 파일만 스테이징한다. 기존 작업 트리에 있던 사용자 변경이나 무관한 포맷 변경을 함께 커밋하지 않는다.
- 제목은 명령형으로 간결하게 작성한다. 예: `feat: add reusable dialog component`, `fix: restore keyboard navigation`, `docs: add contributor agent guidelines`.
